Ghostfolio makes it easy to get your entire investment portfolio in one place. Whether you use the hosted cloud service at [ghostfol.io](https://ghostfol.io) or run it yourself with Docker, you can be tracking your holdings within minutes.

<Tabs>
  <Tab title="Cloud (ghostfol.io)">
    The fastest way to get started is the official **Ghostfolio Premium** cloud offering at [ghostfol.io](https://ghostfol.io). No infrastructure setup required â€” sign up and start tracking immediately.

    <Steps>
      <Step title="Create your account">
        Visit [ghostfol.io](https://ghostfol.io) and click **Get Started**. You can register anonymously or sign in with your Google account.

        <Note>
          The first user to register on a self-hosted Ghostfolio instance automatically receives the `ADMIN` role.
        </Note>
      </Step>
      <Step title="Add a portfolio account">
        Navigate to **My Ghostfolio â†’ Accounts** and click **Add account**. Give it a name (e.g., "Interactive Brokers"), select a currency, and optionally link it to a platform.
      </Step>
      <Step title="Import your activities">
        Go to **My Ghostfolio â†’ Activities** and either add transactions manually or import them in bulk using the REST API or a CSV file.
      </Step>
      <Step title="Explore your portfolio">
        Head to the **Portfolio** section to see your holdings, performance charts, diversification breakdown, and risk analysis.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Self-hosted (Docker)">
    Run Ghostfolio on your own infrastructure using Docker Compose.

    <Steps>
      <Step title="Clone the repository and configure environment">
        ```bash
        git clone https://github.com/ghostfolio/ghostfolio.git
        cd ghostfolio
        cp .env.example .env
        ```
        Edit `.env` and fill in your database credentials, Redis connection, and secret keys.
      </Step>
      <Step title="Start the Docker containers">
        ```bash
        docker compose -f docker/docker-compose.yml up -d
        ```
        This pulls the latest Ghostfolio image from Docker Hub and starts PostgreSQL and Redis alongside it.
      </Step>
      <Step title="Open the application">
        Open [http://localhost:3333](http://localhost:3333) in your browser and click **Get Started** to create your first user account. This first user automatically gets the `ADMIN` role.
      </Step>
      <Step title="Add your portfolio and activities">
        Create accounts, add platforms, and import your trading history (see steps below).
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Import activities via API

If you have existing trading history, you can bulk-import activities using the REST API.

**Step 1 â€” Get a Bearer token:**

```bash
curl -X POST http://localhost:3333/api/v1/auth/anonymous \
  -H "Content-Type: application/json" \
  -d '{"accessToken": "YOUR_SECURITY_TOKEN"}'
```

**Step 2 â€” Import your activities:**

```bash
curl -X POST http://localhost:3333/api/v1/import \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "activities": [
      {
        "currency": "USD",
        "dataSource": "YAHOO",
        "date": "2021-09-15T00:00:00.000Z",
        "fee": 19,
        "quantity": 5,
        "symbol": "MSFT",
        "type": "BUY",
        "unitPrice": 298.58
      }
    ]
  }'
```

| Field | Type | Description |
|-------|------|-------------|
| `accountId` | `string` (optional) | ID of the account to assign the activity to |
| `comment` | `string` (optional) | Free-text comment |
| `currency` | `string` | Currency code, e.g. `USD`, `EUR`, `CHF` |
| `dataSource` | `string` | `YAHOO` \| `COINGECKO` \| `MANUAL` (for custom items) |
| `date` | `string` | ISO-8601 date, e.g. `2021-09-15T00:00:00.000Z` |
| `fee` | `number` | Transaction fee |
| `quantity` | `number` | Number of units |
| `symbol` | `string` | Ticker symbol compatible with the chosen `dataSource` |
| `type` | `string` | `BUY` \| `SELL` \| `DIVIDEND` \| `FEE` \| `INTEREST` \| `ITEM` \| `LIABILITY` |
| `unitPrice` | `number` | Price per unit at time of activity |

A successful import returns `201 Created`. If an activity already exists, the API returns `400 Bad Request` with a duplicate activity error message.
