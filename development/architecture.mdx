Ghostfolio is a full-stack TypeScript application structured as an [Nx](https://nx.dev) monorepo. The workspace is split into two applications (`apps/`) and two shared libraries (`libs/`). A NestJS backend handles all business logic and data access; an Angular frontend delivers the progressive web app experience. PostgreSQL stores all user and portfolio data via Prisma, while Redis provides response caching.

The Nx workspace uses the default project `api` (configured in `nx.json`) with `parallel: 1` to ensure deterministic build ordering. Builds, linting, and tests are all cached by Nx.

## Backend — apps/api

The API is built on [NestJS](https://nestjs.com) and organized into feature modules. Each module is a self-contained directory under `apps/api/src/app/` with its own controller, service, and module file.

### Portfolio calculator

The portfolio module contains a dedicated calculator at `apps/api/src/app/portfolio/calculator/`. It supports four performance calculation methods:

| Directory | Method |
|---|---|
| `mwr/` | Money-Weighted Return (MWR) |
| `roai/` | Return on Average Investment (ROAI) |
| `roi/` | Return on Investment (ROI) |
| `twr/` | Time-Weighted Return (TWR) |

A factory (`portfolio-calculator.factory.ts`) selects the appropriate calculator based on user or system settings.

### API versioning and authentication

All API routes are prefixed with `/api/v1/`. Clients authenticate by passing a Bearer token in the `Authorization` header:

```
Authorization: Bearer eyJh...
```

Obtain a token via `POST /api/v1/auth/anonymous` with the body `{ "accessToken": "<SECURITY_TOKEN>" }`.

The API also supports an impersonation mechanism — access grants between users allow read-restricted or full-read access to another user's portfolio data without sharing credentials.

## Data layer

<CardGroup cols={2}>
  <Card title="PostgreSQL" icon="database">

  </Card>
  <Card title="Prisma ORM" icon="code">
    Schema-first ORM. The canonical schema lives in `prisma/schema.prisma`. Migrations are managed with `prisma migrate dev`.
  </Card>
  <Card title="Redis" icon="bolt">
    In-memory cache for market data, exchange rates, and portfolio calculations. Integrated via `@nestjs/cache-manager` and `@keyv/redis`.
  </Card>
  <Card title="Multiple data providers" icon="plug">
    Market data is sourced from Yahoo Finance, CoinGecko, Alpha Vantage, Financial Modeling Prep, EOD Historical Data, and others — selectable per symbol.
  </Card>
</CardGroup>

### Prisma schema models

The database schema defines these core models:

| Model | Purpose |
|---|---|
| `User` | Application users with roles: `ADMIN`, `USER`, `DEMO`, `INACTIVE` |
| `Account` | Brokerage accounts linked to a user and optional platform |
| `AccountBalance` | Historical cash balance snapshots per account |
| `Order` | Portfolio activities (trades, dividends, fees, etc.) |
| `SymbolProfile` | Cached instrument metadata from data providers |
| `SymbolProfileOverrides` | User-supplied overrides for symbol metadata |
| `MarketData` | Historical price data keyed by data source, date, and symbol |
| `Platform` | Brokerage platforms (e.g. Coinbase, Interactive Brokers) |
| `Tag` | User-defined tags for organizing activities |
| `Access` | Access grants between users for portfolio sharing |
| `Subscription` | Ghostfolio Premium subscription records |
| `Settings` | Per-user JSON settings blob |
| `Analytics` | Request and activity count metrics per user |
| `ApiKey` | Hashed API keys for programmatic access |
| `AuthDevice` | WebAuthn passkey credentials |
| `Property` | Global key-value properties for system configuration |

Authentication providers supported by the `User` model: `ANONYMOUS`, `GOOGLE`, `INTERNET_IDENTITY`.

## Frontend — apps/client

The client is an Angular 19 application built with:

- **[Angular Material](https://material.angular.io)** — component library for UI elements
- **[Bootstrap 4](https://getbootstrap.com)** — utility classes for layout
- **Angular i18n** — internationalization with locale-specific builds (e.g., `/en`, `/de`, `/fr`)
- **Angular Service Worker** — Progressive Web App support

Locale extraction is handled by `npm run extract-locales`, which outputs locale files to `apps/client/src/locales/`.

## CI/CD workflows

The GitHub Actions workflows in `.github/workflows/` automate the following:

| Workflow | File | Purpose |
|---|---|---|
| Build code | `build-code.yml` | Lint, test, and build all affected projects on each push |
| Docker image | `docker-image.yml` | Build and publish multi-arch Docker images to Docker Hub |
| Extract locales | `extract-locales.yml` | Automatically extract and commit i18n translation strings |
